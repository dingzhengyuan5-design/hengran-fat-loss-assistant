# 衡燃 · 在线个性化减脂助手

这是面向18～64岁一般健康成人的静态优先网站。核心方案生成、记录、导入导出可离线使用；配置 Supabase 后可启用端到端加密的匿名同步。

当前版本支持：

- 在同一浏览器中建立多位成员，每位成员拥有独立档案、方案、进度记录和个人菜谱。
- 自动生成具有真实周间变化的4周训练与7天饮食方案。
- 使用120套标准中餐与300种食物进行原料克重、用油量和营养换算，并保存个人或共享菜谱；另含27个中国大陆常见餐饮品牌的486项按份外食场景估算。
- 浏览220项结构化训练动作，其中包含大量徒手动作；每项均提供三阶段教学示意、标准要点、常见错误、呼吸和替代动作，首批88项增加60 FPS浏览器实时插值演示。
- 为4周训练生成具体日期、建议开始/结束时间和当天分段流程，可指定日期、指定星期或采用系统错峰推荐，并可导出 `.ics` 日历。
- 导出当前成员的完整报告，默认包含最近45天进度、完整4周训练和完整7天饮食。

## 本地运行

现在可以直接双击 `index.html` 使用，所有核心交互在 `file://` 下也可运行。若要测试云同步、模拟正式部署或避免浏览器对本地文件的个别限制，建议在本目录启动静态服务器：

```powershell
python -m http.server 8000
```

浏览 `http://127.0.0.1:8000/`。

验证：

```powershell
npm run build
npm test
npm run validate
```

## 动作演示分批审核

正式站点中的 `动作演示审核.html` 可查看当前覆盖状态并逐项打开演示。后续动作批次采用 GitHub 草稿 Pull Request 审核：PR 的 “Checks” 会生成“动作演示审核包”，下载解压后用本地静态服务器打开 `动作演示审核.html`。只有在 PR 中确认通过后才合并，不会按天数自动发布。

## 公网部署

仓库根目录的 GitHub Actions 会验证 `减脂助手网站/` 并发布到 GitHub Pages。需要在仓库 Settings → Pages 中将 Source 设为 GitHub Actions。根目录原有 `index.html` 不会被覆盖。

## Supabase 匿名密文同步

1. 新建 Supabase 项目，执行 `supabase/schema.sql`。
2. 部署 `supabase/functions/profile-sync` Edge Function。
3. 为函数设置 `ALLOWED_ORIGINS`（逗号分隔的正式来源）、`SUPABASE_URL` 与 `SUPABASE_SERVICE_ROLE_KEY`。
4. 复制 `config.example.js` 为 `config.js`，仅填写函数公开地址和正式站点地址。不要把 service role key 写进前端或 Git 仓库。
5. 正式部署时将 CORS 限定到 GitHub Pages/自定义域名。

恢复码在浏览器中派生加密密钥和独立认证令牌；服务端只保存认证令牌的哈希、密文包、同步ID和修订号。恢复码遗失无法找回。

## 数据边界

- 内置300条食物、80套基础套餐、120套标准家庭菜谱和486项外食场景估算，营养值是离线规划快照，不是临床数据库。
- 当前外食条目统一标为配方估算，品牌入口只用于菜单名称核对；它们不是品牌官方营养表。实际门店份量、酱料、糖度和季节配方可能显著改变结果。
- 标准菜谱按明确原料和用油克重计算；餐馆、外卖和用户未称量菜品仍只能估算。
- 单一食物也会因品种、品牌、含水与烹调改变。外卖及混合菜误差更大，界面会显示较低置信度。
- 220项动作示意用于解释动作阶段，不具备摄像头姿态识别或康复评估能力。
- 月度工作流只生成待审核报告和PR，不自动改写健康结论；来源不可达时保留上次快照。
- YouTube指标需要 `YOUTUBE_API_KEY`，且视频ID必须先加入 `data/video-whitelist.json`；B站条目保存人工核验快照；抖音和小红书不使用隐藏接口。

## 目录

- `js/engine.js`：代谢、训练、饮食、趋势与调整规则
- `js/catalog.js`：食物、套餐、文献和视频快照
- `js/recipe-catalog.js`：标准中餐配方、原料克重与营养换算
- `js/exercise-catalog.js`：动作分类、指导要点与教学示意数据
- `js/motion-catalog.js`：动作演示批次、角度模式与审核状态
- `js/external-food-catalog.js`：中国大陆常见外食和茶饮规格估算
- `js/crypto-sync.js`：浏览器加密与同步请求
- `supabase/`：密文表与 Edge Function
- `tests/`：规则与数据验证
